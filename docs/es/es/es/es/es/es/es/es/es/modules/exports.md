# Exportaciones El módulo [Exportaciones](https://omeka.org/s/modules/ExtractText){target=_blank} creará una exportación de información sobre los recursos, activos y más de tu instalación de Omeka S. El módulo puede generar archivos CSV o JSON-LD con metadatos e información interna sobre tus elementos, conjuntos de elementos o medios, así como plantillas de recursos, vocabularios, activos, trabajos, sitios y páginas de sitios, usuarios e información añadida por módulos. (modulesfiles/exports_sidebar.png) ## Requisitos del sistema El módulo requiere dos carpetas dentro de su instalación para poder escribir. Una es una carpeta temporal donde se montan los archivos de exportación, y se establece en su configuración. La otra es el lugar permanente donde se almacenará el resultado de la exportación y estará disponible para su descarga. Esta carpeta estará dentro de la carpeta `/files` de su instalación, llamada "exports". El módulo no creará `/files/exports` en el momento de la instalación, sino cuando inicie su primera exportación. Si encuentra errores, puede que necesite cambiar los permisos de su directorio `/files` para permitir esta creación, o crear manualmente el directorio `/files/exports` y establecer sus permisos. ### Permisos de usuario Este módulo puede ser utilizado por cualquier usuario de nivel Autor y superior. Los usuarios de nivel Investigador pueden ver la tabla de exportaciones anteriores pero no generar nuevas exportaciones. Los usuarios pueden exportar información sobre recursos privados (sitios, páginas, artículos, conjuntos de artículos, etc. sin visibilidad) utilizando este módulo. ## (modulesfiles/exports_config.png) En esta página debe establecer una carpeta en su servidor, dentro del directorio de instalación de Omeka S, para que se creen los archivos temporales de exportación. Introduce la ruta relativa a la ubicación de tu instalación. Debe crear el directorio primero en su servidor, y establecer sus permisos de escritura, antes de guardar esta página. El módulo comprobará que la carpeta existe y que se puede escribir en ella cuando guarde la página, y le dará un mensaje de error si hay algún problema. ## ¡Recursos de exportación![La página de inicio del módulo, recién instalada, mostrando que aún no se ha realizado ninguna exportación](modulesfiles/exports_emptyTable.png) Busca la entrada "Exportaciones" en la sección "Módulos" de la barra lateral, y haz clic en "Crear una exportación" en la esquina superior derecha para configurar tu exportación. la primera página de un proceso de exportación le pide que seleccione "recursos" como tipo de exportación] (modulesfiles/exports_select.png) En este momento, la pantalla sólo ofrece una opción. Es posible que versiones futuras o de terceros de este módulo añadan más opciones de exportación. la segunda página del módulo le permite configurar la exportación para elegir un tipo de información en particular] (modulesfiles/exports_create.png) En la siguiente página configurará su exportación para extraer información de las entradas elegidas en la base de datos de la instalación. **Etiqueta**: Aunque el módulo rellena un nombre por defecto para esta exportación, puedes especificar algo más fácil de recordar, como el tipo de información que estás exportando. **Tipo de recurso**: Seleccione un tipo de información para exportar. Los tres tipos principales de recursos que puede elegir son artículos, conjuntos de artículos o medios. Puede utilizar una consulta para filtrar tipos específicos de estos recursos, como por ejemplo por plantilla de recurso, clase, valor de metadatos, etc. También puede elegir otra información de la base de datos de su instalación, incluidos los datos almacenados por los módulos. Esto puede incluir información sobre importaciones pasadas desde CSV Import o Omeka S Item Importer, jerarquías desde Hierarchy, vocabularios, plantillas de recursos y más. (modulesfiles/exports_resourceTypes.png) Las opciones, excluyendo las opciones adicionales añadidas por los módulos, incluyen: - Tipos primarios: - Ítems - Conjuntos de ítems - Medios - Otros tipos: - Recursos API - Activos - Tipos de datos - Trabajos - Mapeos - Módulos - Propiedades - Clases de recursos - Plantillas de recursos - Recursos - Páginas de sitios - Sitios - Usuarios - Vocabularios. **Consulta de ítems, o consulta de recursos:** Una vez que haya elegido una opción para exportar, puede limitar los resultados con consultas. Puede crear una consulta utilizando el selector de la barra lateral para los recursos (elementos, tipos de elementos y medios). Para otros tipos de recursos, puede introducir manualmente una consulta de texto. Por ejemplo, puede introducir lo siguiente para limitar los recursos según el propietario: ``` owner_id=2 ``` **Formato**: Puede exportar la información a un archivo CSV o JSON-LD (extensiones de archivo `.csv` o `.json`). Cualquiera de las dos opciones vendrá envuelta en un archivo ZIP para su descarga. ** Referencia por**: Este campo es una opción para las exportaciones CSV. Para los recursos, puede rellenar los campos con URLs absolutas (como "https://yourinstallation.org/api/items/123") o sólo con IDs internos. Esto afectará a los campos para archivos, como los campos con derivados de medios (que podrían parecerse a "https://yourinstallation.org/files/large/123abc.png"). **Separador multivalor**: Este campo es una opción para las exportaciones CSV. El carácter por defecto es una tubería vertical (|). Puede cambiarlo si lo desea. Otros separadores comunes son la coma y el punto y coma; se recomienda elegir un carácter que no aparezca en el conjunto de datos. (modulesfiles/exports_inprogress.png) Una vez iniciada la exportación, volverá a la tabla de exportaciones anteriores. Verá una barra verde en la parte superior de la pantalla indicando que el trabajo ha comenzado, con un enlace para ver el estado del trabajo. Si actualiza esta página, verá la tabla con el estado de la exportación indicado. También puede encontrar esta información en la página Trabajos accesible en la barra lateral. ## ¡La tabla de exportaciones![La tabla de exportaciones, con la barra lateral abierta mostrando más información sobre una de las exportaciones pasadas](modulesfiles/exports_sidebar.png) Cuando una exportación esté completa, verá un icono de descarga en la fila de la tabla que, al hacer clic, descargará un archivo ZIP que contiene la exportación que especificó (un archivo CSV o JSON). Una vez creadas las exportaciones, puede ver información sobre ellas haciendo clic en el icono de la elipsis (tres puntos) en la fila de la tabla. Se abrirá una barra lateral en la que podrá revisar la configuración utilizada. Puede volver a ejecutar una exportación con el icono de copia (dos páginas idénticas), o eliminar su registro de la base de datos de la instalación (y cualquier archivo que haya creado) con el icono de eliminación (papelera). Aparecerá una ventana de confirmación. ## Archivos exportados El módulo mostrará todos los metadatos asociados al tipo de recurso elegido, incluyendo información interna como el propietario del elemento y enlaces a todos los medios adjuntos y sus derivados.
