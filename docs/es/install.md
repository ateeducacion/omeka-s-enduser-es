# Instalación ## Requisitos del sistema Para instalar Omeka S (última versión 4.2), necesitará un servidor que ejecute lo siguiente: - Linux
- Apache (con [AllowOverride](https://httpd.apache.org/docs/2.4/mod/core.html#allowoverride){target=_blank} establecido en «All» y [mod_rewrite](http://httpd.apache.org/docs/current/mod/mod_rewrite.html){target=_blank} habilitado)
- MySQL, versión mínima 5.7.9 (o MariaDB, versión mínima 10.2.6) - PHP, versión mínima 8.1, con [PDO](http://php.net/manual/en/intro.pdo.php){target=_blank}, [pdo_mysql](http://php.net/manual/en/ref.pdo-mysql.php){target=_blank} y [xml](http://php.net/manual/en/intro.xml.php){target=_blank} instaladas. 
	- PHP 8.4 no es compatible con Omeka S 4.1 o versiones anteriores, pero sí lo es a partir de Omeka S 4.2 y posteriores. - Opcional, para crear miniaturas: ImageMagick versión 6.7.5 o superior, la extensión PHP `imagick` o la extensión PHP `gd`.

[GD](https://secure.php.net/manual/en/intro.image.php){target=_blank} es una biblioteca gráfica básica que se instala de forma predeterminada con PHP. Solo puede crear miniaturas para formatos de imagen comunes (jpeg, gif, png). [Imagick e ImageMagick](https://www.imagemagick.org){target=_blank} son la misma biblioteca y pueden crear miniaturas para más de 200 formatos. Para obtener más información, consulte la [página de configuración](configuration.md#thumbnails).

## Métodos de instalación ### Instalar desde la descarga !!! nota Antes de instalar Omeka S, debe crear una base de datos MySQL y un usuario. Omeka S debe tener una base de datos dedicada; no puede utilizar un prefijo para una base de datos utilizada por otro sistema o por la instalación de Omeka S o Classic. Para obtener más información sobre cómo crear una base de datos y un usuario, consulte la documentación de soporte de su proveedor de alojamiento o hable con su administrador del sistema.

1. [Descargue la última versión desde la página de versiones](https://omeka.org/s/download/){target=_blank}. 1. Extraiga el archivo zip descargado en su ordenador. 1. Dentro del directorio, abra el archivo `config/database.ini` y añada su nombre de usuario MySQL, contraseña, nombre de la base de datos y nombre del host. El usuario y la base de datos deben crearse antes de este paso.
1. Suba todo este directorio a su servidor, a la carpeta que desee. Por ejemplo, si su servidor es `https://yourwebsite.org/`, es posible que desee instalar Omeka S en una carpeta ubicada en `https://yourwebsite.org/myomekas/`. Asegúrese de que tenga su archivo `database.ini` actualizado (no suba el archivo zip original que descargó).
1. Asegúrese de que Apache pueda escribir en el directorio «files/» del servidor. 1. En su navegador web, vaya a la página «admin» de su instalación de Omeka S, donde podrá completar la instalación. Por ejemplo, si ha subido el contenido del directorio a «https://yourwebsite.org/myomekas/», vaya a «https://yourwebsite.org/myomekas/admin».

### Instalar desde GitHub Las instrucciones básicas para instalar y actualizar desde GitHub se pueden encontrar en el [ReadMe](https://github.com/omeka/omeka-s/blob/develop/README.md){target=_blank} del repositorio GitHub de Omeka S.

A continuación, en su navegador web, vaya a la página de administración de su instalación de Omeka S (`https://yourwebsite.org/myomekas/admin`), donde podrá completar la instalación.

## Configuración inicial Una vez que haya instalado correctamente Omeka S y configurado el archivo `database.ini`, debe navegar a la URL de administración de su instalación de Omeka S (algo así como `https://yourwebsite.org/myomekas/admin`). La primera vez que acceda con su navegador al sitio recién instalado, deberá introducir la información del primer usuario, junto con la información básica de su instalación. Hay dos secciones en esta página: **Crear el primer usuario** y **Configuración**. En la sección **Crear el primer usuario**: - Introduzca una dirección de **correo electrónico** y vuelva a escribirla para confirmarla. - Confirme la **contraseña** y vuelva a escribirla en el siguiente campo para confirmarla. - Introduzca un **nombre de usuario** para el usuario.

Ten en cuenta que puedes cambiar todo esto más adelante en la sección de gestión de [Usuarios](admin/users.md) de tu instalación. ![Sección del primer usuario con los campos descritos](files/installOmekaS1.png) En la sección **Configuración**, introduce: - Un **título de la instalación** que se mostrará en el sitio de administración - La **zona horaria** de la instalación (seleccione en el menú desplegable). - Seleccione una **configuración regional** para el idioma del lado de administración de la instalación. ![Sección de configuración con los campos descritos](files/installOmekaS2.png) Puede cambiar estos datos en cualquier momento en la sección [Configuración](admin/settings.md) de su [panel de administración](admin-dashboard.md).

### Prueba y configura la ruta PHP Omeka S utiliza tareas en segundo plano para algunas tareas de larga duración que operan en muchos elementos o que, de otro modo, podrían llevar mucho tiempo. Omeka S utiliza la CLI (interfaz de línea de comandos) de PHP para ejecutar estas tareas, el comando `php`. Una ruta PHP no válida puede causar una serie de problemas en tu instalación de Omeka Classic. 

Omeka S intentará detectar automáticamente la ruta de acceso a la CLI de PHP en el servidor, pero en algunos servidores esta detección no funciona o hay varios comandos `php` diferentes entre los que elegir. Si [utiliza Reclaim Hosting](#one-click-installation), tendrá que configurar manualmente la ruta de acceso a PHP durante la instalación. Consulte [sus instrucciones aquí](https://support.reclaimhosting.com/hc/en-us/articles/1500005620481#omeka-s){target=_blank}. Utilice la [página de información del sistema](admin-dashboard.md#system-information) para verificar que su instalación ha identificado la ruta PHP correcta. Si se produce un error al hacer clic en el botón, eso puede indicar que debe configurar manualmente. ![Los botones de información del sistema para recuperar la ruta PHP y la versión de ImageMagick.](files/systeminfo_buttons.png) Consulte [Opciones de configuración](configuration.md) para obtener más información sobre cómo modificar la generación de miniaturas, configurar manualmente la ruta PHP y mucho más. 

### Trabajar con su instalación Una vez que haya configurado correctamente todos los componentes técnicos de su instalación de Omeka S, querrá comenzar por: añadir otros usuarios; crear plantillas de recursos, vocabularios y conjuntos de elementos; crear uno o más sitios; y luego añadir elementos y asignar esos recursos a sus sitios. Continúe con el manual del usuario para obtener más información sobre estas partes de Omeka S. !!! nota
	Si ya tiene una instalación de Omeka Classic o S, es posible que desee consultar módulos como el [Omeka Classic Importer](modules/omekaCimporter.md), el [Omeka S Item Importer](modules/ositemimporter.md) o el [CSV Import module](modules/csvimport.md), que pueden ayudarle a copiar otros tipos de datos.

## Actualización ### Actualización manual 1. Descargue la última versión desde la página de versiones. 1. Haga una copia de su directorio `/config`. Tendrá que restaurar sus archivos `local.config.php` y `database.ini` desde esa copia.
1. Haga una copia de su archivo `.htaccess` (en el directorio raíz) si lo ha modificado, por ejemplo, [activando el registro de errores](errorLogging.md). 1. Haga una copia de sus directorios `/modules` y `/themes`. 1. Haga una copia de su directorio `/files`. 1. Elimine todos los archivos de Omeka S y sustitúyalos por los archivos del archivo zip actualizado.
1. Reemplace sus archivos originales `/config/local.config.php`, `/config/database.ini` y `.htaccess`. 1. Reemplace sus directorios originales `/modules`, `/themes` y `/files`. - Para actualizaciones de versión significativas, es posible que también tenga que instalar una versión actualizada de sus módulos y temas. Las notas de la versión del núcleo indicarán si es probable que se produzcan esas actualizaciones. Una vez que complete las migraciones utilizando su navegador, los módulos y temas que requieran nuevas versiones se marcarán claramente en sus respectivas páginas. La página Módulos de su instalación le indicará lo que hay que actualizar; la página Temas de cualquiera de sus sitios indicará los temas activos que hay que actualizar. 1. En su navegador web, vaya a la página de administración de su sitio (`/myomekas/admin`) y ejecute las migraciones que sean necesarias.

### Actualización desde GitHub Las instrucciones básicas para instalar y actualizar desde GitHub se pueden encontrar en el [ReadMe](https://github.com/omeka/omeka-s/blob/develop/README.md){target=_blank} del repositorio GitHub de Omeka S. ## Instalación en Windows o Mac OS (solo para desarrollo)
Omeka S no es compatible con sistemas operativos propietarios o de código cerrado. Sin embargo, para **fines de desarrollo básico** o para una formación rápida, Omeka puede ejecutarse con [WAMP](http://www.wampserver.com){target=_blank}, [MAMP](https://www.mamp.info){target=_blank} o herramientas similares.

Siga las instrucciones de instalación estándar. Deberá realizar los siguientes cambios de [configuración](configuration.md) en el archivo `config/local.config.php` para que funcione.

En primer lugar, es posible que sea necesario establecer la ruta de PHP si Omeka S no puede detectar automáticamente la ubicación de las utilidades de PHP en su servidor local. Abra el archivo `local.config.php` y busque la línea 12: ``` 'cli' => [ 'phpcli_path' => null, ], ```

Rellene el valor `phpcli_path` con la ruta adecuada para su sistema operativo. Por ejemplo, si utiliza un entorno MAMP, encontrará las utilidades PHP dentro de la carpeta de instalación de MAMP en `MAMP\bin\php\php74`. En segundo lugar, debe configurar Omeka S para que utilice la utilidad de generación de miniaturas disponible en su servidor local. Abra el archivo `local.config.php` y busque la siguiente sección:
``` 'service_manager' => [ 'aliases' => [ 'Omeka\File\Store' => 'Omeka\File\Store\Local', 'Omeka\File\Thumbnailer' => 'Omeka\File\Thumbnailer\ImageMagick', ], ], ```
Edite el valor `Omeka\File\Thumbnailer` con lo siguiente, según lo que esté disponible para su sistema: - Reemplace el generador de miniaturas predeterminado por `Omeka\File\Thumbnailer\Gd`.
- Reemplace el generador de miniaturas predeterminado por `Omeka\File\Thumbnailer\Imagick` y habilite Imagick en el archivo `php.ini` de su servidor a través de la interfaz de administración del servidor o directamente en el archivo.
- Mantenga el generador de miniaturas predeterminado `Omeka\File\Thumbnailer\ImageMagick`, pero instale la herramienta de línea de comandos `imagemagick` y cambie el valor `imagemagick_dir` a su directorio. Para instalar imagemagick, consulte la documentación de su servidor. Si ve el error `no input file specified`, es posible que tenga que editar el archivo `.htaccess`. Comente la línea 8, `RewriteRule !\.(php[0-9]?|phtml|phps)$ - [NC,C]`. Es posible que también tenga que comentar las líneas que siguen al bloque de comentarios de la línea 17 («The following rewrites») y sustituirlas por `RewriteRule ^(.*)$ index.php`.
